Interface System Week 3

Interface System Prototype

For this interactive iteration, I wanted to work on some of the changes that I talked about in the last entry. Those changes were having hover effects and changing the hierarchy of sections to make information more clear to the user. This was done by having the sections on the landing page expand by about 25% when the mouse is hovered over the box. This was to make the selection for the user more clear and add some feedback to the user. The hope is to make the landing page a little more enjoyable to the user as well as guide the user behaviour to click on each of the prompts. Although this is a nice feature, I feel that it can only work on desktops as I'm not sure how hovering would work on a phone display. Moreover, I added a lighter color over the text to make sure the legibility of the sections was easier for users. This also helps with hierarchy as it differentiates the title from the pulldown menu. The reason behind this is also to make a signifier to the user that these sections respond differently from the section pulldowns as Norman states the importance of these signifiers. Moving to the sketch and micro project sections, I wanted to add some navigation to help the viewer return to the home menu easier. I did this by implementing some home icons to the bottom of the screen using Font Awesome plugins. Because I was able to add icons, I also decided to add an arrow to make navigation to the different sketches easier. The logic behind this navigation model is because of the rule of having flexibility within the system. I obviously did not add this part to the microproject as there is only one page for this section. I felt that adding the arrow to this screen might cause errors to the site when trying to navigate to a page that does not exist. This logic was because of Neilson’s rule of having a system be free of errors for the user. One thing that I am on the fence about that may help with clarity is having the heading text on the top appear as white rather than black on grey. The micro project and sketch pages still have similar layouts as Shneiderman’s Eight Golden Rules of Interface Design talks about having consistent design that is friendly to users visiting these pages. The mobile design originally differed from the desktop view by having the information centered on the screen. This was done so that fingers won't accidentally press information close to the sides or block the information from being seen. Following the rules of consistent design, I chose to make the desktop view mirror this display of information. Similarly, the added boxes around sections are to make pressing on the pulldown menus easier for fingers to interact with. Having the raw links, like the previous versions, makes the area of interaction much smaller for people on smaller screens.
