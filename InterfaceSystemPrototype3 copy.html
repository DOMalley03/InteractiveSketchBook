Interface System Project

Interface System Statement

In this interactive design project, I was mainly focused on implementing Jakob Neilson’s ‘Visibility of System Status’, which involved making the information more clear. This was done by sorting the information into ‘boxes’ which act as signifiers to the links on the page (affordances). This sorting process greatly improved the navigation model of the sketchbook by removing clutter from the homepage. This also references Neilson’s Aesthetic and Minimalist Design. This navigation model is mirrored on both the desktop version as well as the mobile version. To this, I added a hover function to clearly state to the user what section of the menu they are about to click on which was inspired by Norman’s rule of feedback. Although this addition is in the desktop version, it is not in the mobile version as there is no hover function on the phone. Despite this, the boxes are slightly bigger on mobile so that it is easier to select the different pages in a smaller format. The format stays the same which also is in reference to Neilson’s Consistency and Standards (Shneiderman’s strive for consistency). The hierarchy of the information was also important for this project as I did not want the user to get confused when clicking on a section. There is bolder text for the titles of each section on the landing page, with smaller and unbolded text for the individual names of the webpages. The titles of the sections were also re-organized from being a weekly format to a more clear section format. Similarly, the pages of the sketches and micro-project have a clear title, with the sketch directly underneath the title. Below the sketch is the journal entries as the sketch should be the main feature of the page. This also helps with the mobile-first development as the user doesn't have to scroll down to access the important information. Similarly, I also added functions to the bottom of the screens to quickly navigate to the landing page, the next page, and the previous page (thanks john!). This helps with the short-term memory load of the user as they don’t have to navigate all the way back to the home page each time they want to access a new sketch. This also helps with users who want to go back to the landing page after accessing a number of sketches which was done to give the user more control of the system (Shneiderman). Because many phone apps are created with this style of navigation, it should feel more intuitive for phone users to navigate between the different sketches using the icons. Before, a phone user would have to find the built in browser function to go back to the previous page, which might have been a previous sketch rather than the landing page. The layout of both the sketches and the micro-project page are similar as I wanted to keep a consistent layout between the webpages. Overall, I believe the goal of organizing information and making the site friendlier to mobile-users was achieved through this figma project.
