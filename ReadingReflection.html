Reading Reflection

Norman Chapter 1

After reviewing the chapter, I believe that my button game has evolved to incorporate some of these design principles. The most obvious affordance that is present in my micro project is the word “PUSH” that is placed on the big red button. This acts as a signifier/ cue that the button can be pressed by the user. Similarly, the color of the multiplier upgrade button helps it stand out from the rest of the screen prompting the user that it can be pressed as well. Although I think these are clear indicators, there were a few parts of my project where affordances were unclear or hidden. One that was hidden was the multiplier meter that slowly goes down over time. Although there is a little counter at the top to represent the countdown, adding a bar or making the indicator bigger might help with clarity to the user. The addition of the counter was to force the user to keep clicking otherwise they would lose their hard earned upgrades. Although there is some feedback in my micro project, I could have added more to make it even more clear to the user. When playing the game, the button in the middle will depress as if the user is actually pressing the button. This indicates to the user that their input was received and the game is acting in accordance to their action. Although it is a small animation, I personally feel that button press is the most important part of my project. One feedback that I felt I could have added was sound to indicate that the button was pressed. The sound paired with the animation of the button helps solidify that the user’s input was received and also would make the game more enjoyable to play. I think that the input and output of the project is pretty intuitive. The start screen informs the user that the goal is to get as many points in one minute. When the button is pressed a point is added to the counter. The upgrade button also has a number that correlates to how many points needed to get that next boost. One part that may be an example of intentional confusion is when the button starts to move around after reaching a multiplier higher than three. Although it may be confusing seeing the button move around, the concept of pressing the button is still the same. There are two examples of constraints within my project. The first constraint is the hard timer that is started when the game starts. This counts down for one minute, after which the game is ended and the score is shown. This does not change at any point and guides the user to the intended playstyle. The second one that is arguably not as clear is the multiplier countdown. This is more invisible as it only shows up after reaching the multiplier threshold. There are some aspects of the design that may not be as clear to the user, especially if Norman critiqued it. One would be the hierarchy of information as most of the counters live on the top left corner of the project. Possibly having the indicators be visible shapes rather than numerical values would help with clarity, especially playing a game that requires focus on the button. I enjoyed the reading because it gave context to some of the ideas that we discussed in class. I think knowing the vocabulary before entering this next unit is important for understanding aspects of interactive design; like affordances for possible actions and signifiers for cues.
