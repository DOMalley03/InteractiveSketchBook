Concept Summary

I want to create a cookie-clicker like game where the user clicks a button on screen. When the button is pressed a currency counter goes up. This money from clicking can be used to purchase an upgrade that makes getting more currency easier.

Variables and Input

let points = 0; Points would be the main currency of the game and clicking the button adds one point. This changes behavior by enticing the user to get a higher score. Buying the upgrades would also make a multiplier go up which can be: let mult = 1

Base Build

let points = 0; let mult = 1; when function mousePressed is used, let points += mult; This should add the multiplier on each click so that clicking now gives the multiplier amount if upgrade is bought, spend 25 points and mult++. This should add one to the multiplier making clicking easier

Structural Growth

Some ideas that I have to grow this project would be possibly adding an 'auto-clicker' upgrade that would automatically give points every second. This could run on a timer variable. Another idea would be to add a decay factor that takes away points every second. It would be nice to also add a possible animation to the click to make it more alive.

March 2nd

Current Build

I was able to get the multiplier working as well as the timer for the 'combo'. The main variable is still the points, but I believe the multiplier is what makes the button fun to click. When the user clicks the multiplier it makes it easier to click. The catch now is you must click within 5 seconds or you lose progress on the multiplier.

Pressure

There is now a timer that starts once the user hits a 3x multiplier. This foces the user to click within 5 seconds or the multiplier will go back down to 0. This adds pressure to the game and rewards constant interaction with the button.

Future Additions

I would still like to add an auto clikcer function that adds a point every second. I dont believe this would interact with the multiplier as the timer resets when the mouse is clicked. I think adding the auto clicker would help with getting through levels faster and also make it less tedious to click.

March 22nd

Self Assessment

Core Rule & Clarity

20pts -The core rule is that the button must be pressed by the mouse to increase the score. Although there are upgrades and button movement, the button must be pressed to increase score

State Progression

15pts - I didnt want to add levels to mine so I decided to make the button harder to press as it continues. After the multiplier hits 3, the button will start to move and a 'combo' timer will count down for 3 seconds. if the button is not pressed in this time the multiplier will be lost and must be built up again.

Interaction/Evolution

15pts - At first I would say there was not much interaction. It initially started as a button that did not react or move, with the only indicator being a tiny score going up. Adding the multiplier and the timers gives a sense of urgency for the minute of clicking. Moreover, adding the start and finish screens also allows a break for the user to admire their score.

Iterative Development

15pts - I think compared to where this started there was a lot of evolution. At first it was a hard-coded circle that remained static. Implementing the movement forced me to change all those hard-coded positions with variables that changed randomly. This was probably the hardest part of the project and I now know not to hard code something like that. I think adding the multiplier instead of the auto clicker function also makes the game more interactive rather than being another idle cookie clicker game. Similarly, I was initially going to have the score keep going up. I think having the timer for one minute makes the game more of a challenge, especially when the button starts moving around.
