Interface Audit

Micro-Project Audit

For this interactive audit I will be analyzing my micro project page. I believe that my microproject sketch is unclear with communicating to the user where exactly they are. Although the sketch has a start screen with instructions, it is not stated that the user is about to play a game. I believe moving the text below the sketch and adding a title might make the intention more clear. Actions that are available on the page include pressing the spacebar to start the game and the buttons to click once the game has started. The instructions on the start screen prompts the user to press the spacebar so I believe that is clear. There is, however, no instructions for pressing the buttons besides the word “push” centered on the red circle and “upgrade” on the green button. The interaction of clicking the button is solidified by having the button depress when the user clicks. I believe these signifiers (color, words, and animation) make the affordances of clicking the button more intuitive. I think having the text above the game makes the navigation to the game a little unclear. Because the user most likely will start from the top and scroll down, the journal entries are the first thing they will see. I believe the journal entries are clear in their hierarchy as they use a bolder font and larger size. The game itself may not be predictable as the button will start to move around, but this is part of the game design so I’m not sure if that counts. Although this part may be unclear, I think the larger red button and smaller green one is clear in its hierarchy. The red button is centered and several times larger than the green button which is off to the side of the screen. A user may not understand the value of the upgrade button as it lays off to the side which may cause them not to use it. I think the page assumes the user understands that the sketch is a game, although there is no indication that it is one. One part that requires guessing is the multiplier meter. This part of the game is not explained, but reinforced by the countdown timer that starts after hitting a 3x multiplier. This restricts the user from pressing the button or risk losing their multiplier; which is also something that the game assumes the user can figure out. The hard timer of one minute also restricts the user into only interacting with the sketch for that period. One part of my sketch that violates Jakob Neilson’s rule of user control and freedom is that there is no restart or reset button. Once the spacebar is pressed, the minute timer starts and the user is locked into playing the game (besides refreshing the page). I think adding a reset button or pressing the key “R” may feel more freeing to users that might want to re-read the start menu. Another rule that my sketch violates is the aesthetic and minimalist design. I think the sketch could be improved by having less words for some of the counters. Instead of having a number for timer, multiplier, and mult timer, I could have made use of a bar that shrinks over time. Because the user is more focused on playing the game, having a visual signifier rather than a numerical one might be more clear to the user in that instance.
